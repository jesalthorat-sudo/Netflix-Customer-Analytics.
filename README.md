Netflix Customer Churn & Revenue Analytics

Project Overview

This project analyzes customer subscription behavior, revenue performance, customer engagement, and churn patterns using MySQL. The objective was to derive business insights from customer data and build a foundation for an interactive Tableau dashboard.

Business Problem

Subscription-based businesses need to understand customer behavior, identify churn risks, and analyze revenue drivers. This project focuses on answering key business questions such as:

* What is the customer churn rate?
* Which subscription plans generate the most revenue?
* Which cities contribute the highest revenue?
* Who are the most engaged customers?
* How can customers be segmented based on watch behavior?

Dataset

A customer dataset was created and expanded to 640+ records using SQL. The dataset contains:

* Customer ID
* Customer Name
* City
* Subscription Plan
* Monthly Fee
* Watch Hours
* Customer Status (Active/Churned)

SQL Concepts Used

* SELECT Statements
* Filtering with WHERE
* GROUP BY and ORDER BY
* Aggregate Functions (COUNT, SUM, AVG)
* CASE WHEN Statements
* Views
* Common Table Expressions (CTEs)
* Window Functions
* RANK()
* DENSE_RANK()

Key Analysis Performed

Customer KPIs

* Total Customers
* Total Revenue
* Average Watch Hours
* Customer Churn Rate

Revenue Analysis

* Revenue by City
* Revenue by Subscription Plan
* Revenue Contribution Percentage

Customer Analytics

* Customer Segmentation
* Top Customers by Watch Hours
* Active vs Churned Customer Analysis

Key Insights

* Identified the highest revenue-generating cities.
* Measured customer churn and its impact on revenue.
* Segmented customers into Heavy, Medium, and Low Users based on engagement.
* Ranked customers based on watch hours and subscription value.
* Generated business KPIs for executive reporting.

Tools Used

* MySQL 8.0
* MySQL Workbench
* Tableau Public (Dashboard Visualization)

Project Outcome

Built an end-to-end customer analytics solution using SQL to generate business insights and support data-driven decision-making. The project demonstrates practical use of advanced SQL techniques commonly used in real-world analytics environments.
